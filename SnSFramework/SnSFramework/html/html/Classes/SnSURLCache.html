getFromCache:cachePolicy:timeoutInterval:

getFromCache:timeoutInterval:andDelegate:

Retrieves some data from the cache in two phases: the data is returned from the cache if available, and then updated from Internet.

- (NSData *)getFromCache:(NSString *)url timeoutInterval:(NSTimeInterval)timeoutInterval andDelegate:(SnSDelegateWithObject *)delegate

Parameters

delegate

the object that will trigger its underlying selector when the data have been updated from Internet. The selector must have the following signature: (void) selector:(SnSURLCacheData *)urlCacheData

Return Value

the cached data; may be nil if the cache does not already contain the data @throws SnSURLCacheException if the data is not available according to the cache policy asked for

Discussion

Retrieves some data from the cache in two phases: the data is returned from the cache if available, and then updated from Internet.

The method returns as soon as the potential local cache has been retrieved, and will perform the data reloading from Internet in a detached separate thread.
